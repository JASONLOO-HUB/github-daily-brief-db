## 给 AI 产品经理的模式信号

1. **桌面 Agent 需要“看见界面 + 能安全点按 + 可追踪动作”**：UI-TARS-desktop 的价值不是另一个聊天框，而是把视觉理解、GUI 操作和任务编排合成可执行栈。
2. **垂直行业 demo 正在成为大模型销售材料**：financial-services 显示 Anthropic 在用开源示例把金融场景拆成可演示 workflow，值得 PM 研究“行业模板如何降低采购理解成本”。
3. **中文 AI 学习产品进入项目制阶段**：easy-vibe 不是单点工具，而是用 vibe coding 课程把学习者拉进可交付项目，适合转行内容和训练营产品。
4. **个人 AI 工具需要本地推理和记忆层**：omlx 解决本地模型服务，Tolaria/Hermes 解决长期上下文，组合后就是个人工作台的底层能力。
5. **Skill/harness 会成为 Agent 产品的可复用资产层**：deepagents 和 agent-skills 分别对应架构层与 SOP 层，说明未来竞争点在“怎样稳定复现复杂任务”。

## 小红书最值得做的 3 个选题方向

- **方向 1：AI 不是只能聊天，它已经会操作桌面**
  标题钩子：*“真正的 AI 助理不是回答你，而是能帮你点开软件干活”*。
  脚本钩子：用 UI-TARS-desktop 拆“看屏幕、理解按钮、执行步骤、回传结果”四段。

- **方向 2：文科生学 AI 编程，不要先背语法，先做项目**
  标题钩子：*“我会让 AI 写代码，但我真正学的是怎么验收结果”*。
  脚本钩子：用 easy-vibe、agent-skills 讲“任务卡、验收标准、修改迭代”。

- **方向 3：为什么 AI 做长任务总忘事？因为它缺记忆系统**
  标题钩子：*“AI 不是笨，是你每次都让它失忆重开”*。
  脚本钩子：用 Tolaria、Hermes Agent 解释长期记忆、任务恢复、知识库与上下文管理。

## 今日最值得产品化尝试的 1-2 个机会

**机会 A：中文 Agent Skill + 行业模板库**
MVP：把 financial-services、agent-skills、deepagents 的公开案例拆成中文任务卡：输入材料、执行步骤、模型/工具依赖、验收标准、风险提示。
目标用户：AI 产品经理、转行学习者、企业内训讲师。
差异化：不是 prompt 合集，而是“可演示、可复盘、可评估”的工作流模板。
风险：行业案例容易被过度包装，必须标注 demo 边界和合规限制。

**机会 B：GitHub 热门项目到小红书脚本的素材雷达**
MVP：每天抓取 GitHub Explore/Trending/Release，自动输出 3 个标题、30 秒口播、项目截图清单、适合人群和风险提醒。
目标用户：AI 教育账号、小红书矩阵号、AI 工具社群。
差异化：把开发者信号翻译成中文学习路径和产品机会，而不是简单搬运仓库名。
风险：需要事实/推断分离，避免把短期 star 增长误判为成熟产品。
