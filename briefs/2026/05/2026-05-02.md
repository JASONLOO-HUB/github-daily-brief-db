## 今日判断（摘要）

今天的主线不是单个 Agent 更强，而是 Agent 正在获得“可迁移的工作规程”和“可交付的视觉资产”。新近走红项目集中在 Agent Skills、DESIGN.md、开源设计工作台、PPT Skill 和 ML 研究自动化；已有热门项目则继续围绕 coding agent 的权限、沙箱、工作区、远程控制和开源治理迭代。

## 给 AI 产品经理的模式信号

1. **Skill 正在成为 Agent 产品的可复用单位**：open-design、agent-skills、guizang-ppt-skill 都不是新模型，而是把可重复流程封装成规则、资源和验收方式。
2. **设计上下文开始像工程上下文一样被版本化**：DESIGN.md 的价值在于把颜色、字体、组件、风格理由放进仓库，让 Agent 不再每次“凭感觉设计”。
3. **coding agent 的竞争点转向治理能力**：Codex、Claude Code、opencode、goose 都在处理权限、沙箱、工作区、远程控制、插件、MCP/ACP 接入等交付问题。
4. **内容生产工具从 prompt 走向交付物**：PPT Skill、Open Design 这类项目更适合讲“可打开、可修改、可复用”的办公资产，而不是只展示生成效果。

## 小红书最值得做的 3 个选题方向

- **方向 1：AI 设计终于有说明书了**
  标题钩子：*“为什么 AI 做 UI 总像模板？因为你没给它一本 DESIGN.md”*。
  脚本钩子：用 DESIGN.md 解释“品牌规则、设计 token、风格理由”如何变成 Agent 可执行的上下文。

- **方向 2：Skill 才是普通人驾驭 Agent 的关键**
  标题钩子：*“别再只收藏提示词了，下一步要收藏 AI 技能包”*。
  脚本钩子：对比 agent-skills 和 guizang-ppt-skill：一个管工程质量，一个管 PPT 风格。

- **方向 3：开源 AI PPT 的重点不是炫，而是能交付**
  标题钩子：*“一个中文创作者开源的 PPT Skill，为什么能冲上 GitHub？”*。
  脚本钩子：拆解 10 个固定布局、5 套主题、单文件 HTML 输出，说明约束如何降低审美波动。

## 今日最值得产品化尝试的 1-2 个机会

**机会 A：中文 Agent Skill 模板市场**
MVP：收集 30 个适合 AI 产品/运营/内容创作者的 Skill，包括选题研究、PPT 生成、竞品拆解、PRD 草拟、数据表清洗；每个 Skill 配“适用工具、输入模板、输出样例、风险边界、验收清单”。
目标用户：AI 转行学习者、小红书博主、独立顾问、小团队产品运营。
差异化：不是卖 prompt，而是卖“可复现流程 + 交付标准 + 中文场景样例”。
风险：Skill 标准和各 Agent 适配仍在变化，需要持续维护版本兼容。

**机会 B：DESIGN.md + PPT Skill 的个人品牌视觉包**
MVP：用户上传小红书主页、PPT 样例或品牌色，生成一个 DESIGN.md、PPT Skill 主题和 3 套内容模板。
目标用户：知识博主、AI 课程讲师、咨询顾问、求职作品集用户。
差异化：把“审美建议”变成 Agent 可执行文件，让后续图文、PPT、落地页保持风格一致。
风险：版权素材、品牌侵权和审美同质化需要明确提示。
