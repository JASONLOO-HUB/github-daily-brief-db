## 给 AI 产品经理的模式信号

1. **RAG 重新回到“证据结构”问题**：PageIndex 的热度说明用户不只要把文档塞进向量库，而是要表格、图文、章节和引用能被稳定定位。
2. **视觉理解成为内容生产前置层**：Understand-Anything 适合做素材理解、课程拆解、视频知识点提取，不只是图像问答 demo。
3. **Skills 正在从工程师工具变成产品机制**：taste-skill 和 agent-skills 说明“经验、审美、流程”都可以被封装成 AI 可调用的上下文包。
4. **设计和研究都在走 artifact-first**：open-design 输出可改设计稿，deer-flow 输出带来源的研究过程，重点都是可检查成果。
5. **Agent 产品需要后台运行层**：ruflo 的价值不在聊天界面，而在任务状态、daemon、插件和失败处理。

## 小红书最值得做的 3 个选题方向

- **方向 1：别再只收藏 prompt，真正值钱的是 AI Skill**
  标题钩子：*“高手不是会写 prompt，而是会给 AI 装工作流程”*。
  脚本钩子：用 taste-skill 和 agent-skills 解释 Skill = 规则 + 示例 + 验收标准 + 可复用文件夹。

- **方向 2：AI 读论文/报告，不能只靠向量库**
  标题钩子：*“为什么 AI 读 PDF 经常胡说？因为它没读懂表格和章节结构”*。
  脚本钩子：用 PageIndex 讲“文档索引、证据定位、推理检索”三步。

- **方向 3：普通人也能做 deep research 工作流**
  标题钩子：*“AI 做调研不是一键生成，而是像实习生一样找证据、列提纲、写报告”*。
  脚本钩子：用 deer-flow、ruflo、DeepSeek-TUI 拆“研究、执行、记录”的完整链路。

## 今日最值得产品化尝试的 1-2 个机会

**机会 A：中文 AI Skill 市场 + 任务卡生成器**
MVP：收集 GitHub 上高热 Skills，转成中文任务卡，标注适配工具、输入材料、执行步骤、验收标准和风险。
目标用户：AI 产品经理、内容创作者、AI 转行学习者、知识付费社群。
差异化：不是 prompt 合集，而是“可安装、可复用、可验证”的工作流资产。
风险：不同 Agent 的 Skill 协议和触发方式不统一，必须逐项标注适配范围。

**机会 B：GitHub 热门项目到课程/小红书脚本的证据库**
MVP：输入 repo URL，自动抽取事实、来源、非工程解释、PRD 灵感、标题和 60 秒脚本，并维护按日期的变化快照。
目标用户：AI 教育账号、课程讲师、AI 产品经理。
差异化：把 GitHub 信息差转成可讲、可演示、可复盘的内容资产。
风险：需要事实/推断分离；不能把短期 star 热度包装成成熟产品。
