## 给 AI 产品经理的模式信号

1. **Agent 入口正在从聊天框转向“工作环境”**：DeepSeek-TUI 抢终端，browserbase/skills 抢浏览器，context-mode 抢上下文和记忆。
2. **Skills 市场正在分化**：mattpocock/skills 偏工程方法论，ComposioHQ/awesome-codex-skills 偏可执行任务和外部应用连接。
3. **本地优先成为 deep research 的反向卖点**：local-deep-research 的价值不是更炫，而是让资料、模型、资源限制和隐私更可控。
4. **内容工具开始强调整条流水线**：Pixelle-Video 把脚本、配图/视频、TTS、BGM、合成和 Web UI 串起来，接近创作者真实工作。
5. **基础设施项目在补“可运行性”**：Ruflo 和 cocoindex 的更新都不是纯概念，而是围绕 daemon、插件、连接器、图数据、测试和文档。

  脚本钩子：用 mattpocock/skills 和 ComposioHQ/awesome-codex-skills 讲 Skills = 工作流程 + 示例 + 验收标准 + 可复用目录。

## 今日最值得产品化尝试的 1-2 个机会

**机会 A：中文 Agent Skills 导航 + 安装助手**
MVP：抓取 GitHub 上高热 Skills 仓库，按“产品经理、运营、内容、研发、求职”转译成中文任务卡，提供适配 Codex/Claude 的安装说明和验证清单。
目标用户：AI 转行学习者、产品经理、小红书 AI 教育账号、知识付费社群。
差异化：不是 prompt 合集，而是“技能用途、适配工具、执行步骤、验收标准、风险提示”。
风险：不同 Agent 的 skills/plugin 协议差异大，必须标注适配范围，避免过度承诺。

**机会 B：GitHub 热门项目到内容流水线的选题工具**

目标用户：AI 教育博主、课程讲师、AI 产品经理。
差异化：把开源项目信号转成可讲、可演示、可做社群作业的内容资产。
