## 给 AI 产品经理的模式信号

1. **个人 Agent 正在产品化为“会记住的数字员工”**：openhuman 和 agentmemory 都说明，用户不只要一次性回答，还要跨会话身份、偏好、项目背景和任务历史。
2. **行业插件是大模型落地的低摩擦形态**：claude-for-legal 把法律工作拆成可安装能力，说明垂直行业 PM 可以先做插件/模板，再谈完整平台。
3. **多模型时代需要路由和成本治理**：OrcaRouter-Lite 的 BYOK、OpenAI-compatible API 和兜底模型思路，适合做企业 AI 中台的轻量入口。
4. **Agent 生态需要协议和可观测性**：open-multi-agent、AgentsMesh、future-agi 分别对应通信协议、运行框架、评测监控，说明“能跑起来”之后马上会遇到协同和质量问题。
5. **AI coding 进入“规范 + Skill + 设计上下文”阶段**：spec-kit、mattpocock/skills、awesome-design-md 共同指向一个趋势：让 AI 写代码前，先给它需求规格、团队 SOP 和设计规则。

## 小红书最值得做的 3 个选题方向

- **方向 1：AI 助理为什么总失忆？**
  标题钩子：*“不是 AI 不聪明，是你每次都让它从零认识你。”*
  脚本钩子：用 openhuman 和 agentmemory 拆“长期记忆、偏好、项目背景、回忆检索”的概念。

- **方向 2：文科生也能理解的 AI 团队协作**
  标题钩子：*“一个 AI 干不了所有活，以后是多个 AI 分工合作。”*
  脚本钩子：把 open-multi-agent 和 AgentsMesh 类比成“部门群聊 + 流程系统 + 交接记录”。

- **方向 3：AI 写代码前，先写清楚需求**
  标题钩子：*“别再直接叫 AI 写页面了，先给它一份产品说明书。”*
  脚本钩子：用 spec-kit、awesome-design-md、mattpocock/skills 讲 PRD、设计规范和工作 SOP 如何减少返工。

## 今日最值得产品化尝试的 1-2 个机会

**机会 A：中文 Agent 记忆与 SOP 工作台**
MVP：为个人创作者或小团队提供一个轻量库，记录账号定位、风格偏好、选题历史、禁用表达、常用流程，并导出给 Claude/Codex/Gemini CLI 的上下文包。
目标用户：小红书 AI 教育账号、独立顾问、AI 产品经理。
差异化：不是 prompt 合集，而是“记忆 + SOP + 验收清单 + 版本历史”。
风险：隐私和账号数据敏感，需要本地优先、可删除、可审计。

**机会 B：GitHub 热门项目到 PRD/spec 的自动翻译器**
MVP：每天抓热门 repo，自动生成中文项目卡、目标用户、核心流程、PRD 假设、验收标准和可复刻 demo。
目标用户：转行 AI 产品经理、AI 训练营、内容社群。
差异化：把开发者 README 翻译成产品需求与课程作业，而不是只搬运项目介绍。
风险：repo 热度不等于可用性，必须保留来源、成熟度和推断边界。
