## 给 AI 产品经理的模式信号

1. **Agent orchestration 正在产品化**：Symphony、OpenHands、Dify 都在解决“任务如何被派发、执行、验收、协作”的问题。PM 需要开始设计 agent 工作台，而不是只设计 prompt 输入框。
2. **记忆层成为新基础设施**：MemPalace 说明“用户历史、项目上下文、偏好和关系图谱”会成为 agent 产品的核心护城河。
3. **工具调用正在从 API 扩展到真实软件环境**：agent-browser 和 googleworkspace/cli 都把现有软件包装成 agent 可操作界面，这对“不会写代码的人如何自动化日常工作”很关键。
4. **安全和权限会成为差异化**：Composio 的更新把文件自动上传下载改成显式 opt-in，说明 agent 工具链的默认行为必须可解释、可授权、可回滚。

## 产品化机会

**机会 A：GitHub AI 信息差选题雷达**
MVP：每天抓 GitHub 热门 AI repo，自动输出中文解释、受众标签、选题标题、脚本钩子和风险提示。
目标用户：AI 内容创作者、转行学习者、AI 产品经理。
差异化：不是翻译 README，而是把开源项目转成“学习路径 + 内容选题 + 产品机会”。
风险：GitHub 热度容易被 star 噪声污染，需要引入 release、commit、issue、外部讨论做交叉验证。

**机会 B：文科生 AI 自动化任务库**
MVP：把 agent-browser、Google Workspace CLI、Dify 这类项目包装成“每天一个可复制工作流”，例如自动整理简历、抓招聘信息、生成选题日历。
目标用户：不会写代码但愿意用 AI 提升学习和求职效率的人。
差异化：从“工具推荐”升级为“可运行的任务模板”。
风险：真实账号、浏览器和文件权限涉及隐私，需要强权限提示和本地优先设计。
