# langgenius/dify

## 稳定定位

Dify 是开源 LLM 应用开发平台，覆盖 workflow、RAG、agent、模型管理、可观测和 API 化交付，定位从原型到生产。

## 原理解释

Dify 把 AI 应用常见能力放到一个平台：可视化 workflow 负责流程编排，RAG pipeline 负责知识检索，agent tools 负责外部动作，模型管理负责多模型接入，LLMOps 负责日志和性能分析。它更像 AI 应用操作系统，而不是单个聊天应用。

## 典型用例

- 产品团队用可视化 workflow 快速搭建客服、知识库、内容生成工具。
- 公司自托管 Dify，连接内部知识库和模型供应商。
- PM 用 Dify 验证 agentic workflow 的产品原型。

## 对 AI 产品经理的价值

Dify 是观察 AI 应用平台化的样本。PM 可以借鉴它的模块边界：workflow、RAG、model provider、tool、plugin、observability、billing、collaboration、security。

## 对小红书账号的价值

适合讲“AI 产品不是一个 prompt，而是一套流程”：用户输入、知识库、工具、模型、输出、监控都要设计。

## 产品化机会

用 Dify 思路做“中文 AI 应用模板市场”：为文科生提供求职、学习、写作、选题、资料整理等模板，并把复杂配置隐藏在教学内容后面。

## 风险与限制

Dify 功能重，普通用户自托管和配置成本高。对小红书受众更适合先讲产品结构和模板价值，不宜一开始要求完整部署。

## 2026-05-01 长期档案更新

事实：本轮继续把 Dify 作为 major_update 观察对象，因为它代表 workflow、RAG、Agent、模型接入和 LLMOps 平台化路线。
推断：当更多入口型 Agent 项目走热时，Dify 的长期价值更像“AI 应用中台”，适合被拆成课程和模板市场，而不是单独作为聊天工具讲解。
