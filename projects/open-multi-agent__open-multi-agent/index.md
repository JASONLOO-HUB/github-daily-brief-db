# open-multi-agent/open-multi-agent

## 稳定定位

open-multi-agent 是面向多 Agent 协作的开放协议和 TypeScript SDK，目标是让不同 Agent 之间能用更标准的方式发现、通信、交接任务和共享状态。

## 原理解释

一个 Agent 像一个员工，多 Agent 系统像团队。团队要协作就需要协议：谁负责什么、怎么发消息、如何交接上下文、失败谁处理。open-multi-agent 试图把这些规则做成开发者可接入的接口。

## 典型用例

- 把研究 Agent、写作 Agent、代码 Agent 串成一个工作流。
- 给企业内部不同 AI 助理建立统一通信方式。
- 做多 Agent demo，观察角色分工和任务交接质量。

## 对 AI 产品经理的价值

它提示 PM：多 Agent 产品不是“多放几个机器人”，而是需要任务路由、状态同步、权限和可追踪记录。PRD 应明确协作边界和失败交接机制。

## 对小红书账号的价值

适合用“AI 团队分工”解释复杂 Agent：一个负责调研，一个负责写作，一个负责检查，最后由协调者汇总。

## 产品化机会

MVP 可以做“多 Agent 任务卡模板”：定义角色、输入、输出、交接字段、检查点和人工确认点。

## 风险与限制

协议类项目价值取决于生态采用率。过早押注单一协议有风险，应关注兼容性和可替换性。
