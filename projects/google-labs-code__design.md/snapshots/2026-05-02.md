# 2026-05-02 Snapshot：google-labs-code/design.md

## 为什么今天值得关注

**事实**：仓库提供 DESIGN.md 模板和示例，强调把设计规则写成 AI 可读上下文；第三方趋势页显示其短期关注度上升。

**推断**：当 Agent 能直接改页面、写组件、做 PPT 时，设计上下文会像 README、AGENTS.md 一样成为项目基础设施。

## 当天变化与信号

- new_hot：主题从 prompt 扩展到 context engineering 和 design system。
- PM 信号：AI 产品要支持用户把偏好、限制和品牌规则固化为可复用文件。
- 内容信号：适合用“AI 不懂你的审美，是因为你没有给它规则”做教育选题。

## 典型用例

- 网站改版前先生成 DESIGN.md，再让 Agent 统一修改页面。
- 小红书账号做个人视觉规范，减少每篇封面风格漂移。
- 团队把设计原则和禁用样式放进仓库，降低 AI 生成 UI 的返工。

## 产品化机会

MVP：上传 5 张历史封面或页面截图，生成 DESIGN.md 草稿和可复制的 Claude/Codex 项目说明。

## 风险与限制

规范需要真实样例和人工判断。只用空泛形容词会让 Agent 继续输出模板化设计。
