# Leonxlnx/taste-skill

## 稳定定位

taste-skill 是一个把设计审美、视觉判断和创作偏好沉淀成 Agent Skill 的项目。它的价值不在模型本身，而在把“好看与不好看”的经验写成 AI 可复用上下文。

## 原理解释

很多人以为 AI 设计差是因为 prompt 不够长。taste-skill 的思路更像给 AI 一本“审美手册”：什么样的排版更克制，什么颜色容易廉价，什么元素会显得模板化。Agent 在生成设计时先读这套规则，再输出更接近人类审美的结果。

## 典型用例

- 设计 Agent 生成封面、网页、海报前先加载审美规则。
- 内容团队把自己的品牌偏好、禁用风格和示例图整理成 Skill。
- AI 教育账号用它解释“审美也可以被工程化为规则和案例”。

## 对 AI 产品经理的价值

它把“风格质量”产品化为可管理资产：风格规则、反例、示例、触发条件、版本管理。PM 可借鉴到品牌助手、内容审核、设计工作台和企业模板系统。

## 对小红书账号的价值

非常适合讲“为什么 AI 做图总有网感廉价味”。切入角度是：不是多写几个形容词，而是给 AI 建一个可复用的审美标准库。

## 产品化机会

MVP：小红书账号审美 Skill 生成器。用户上传 5 张喜欢/不喜欢的图，生成一份中文风格手册、封面模板规则和 Agent 可读 Skill。
目标用户：小红书博主、设计新手、AI 课程社群。
差异化：把抽象审美转成可复用规则，不只卖模板。

## 风险与限制

审美判断主观性强，不能保证所有场景都适配。license 未在本轮页面采集中明确确认，复用前需要二次核验版权和授权。
