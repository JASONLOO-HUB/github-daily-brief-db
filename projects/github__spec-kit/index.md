# github/spec-kit

## 稳定定位

spec-kit 是 GitHub 推动的 spec-driven development 工具包，目标是让产品规格、实现计划和代码任务更结构化地连接起来，尤其适合 AI coding agent 参与开发流程。

## 原理解释

直接让 AI 写代码，容易出现“看似完成、其实偏题”。spec-driven development 先写清楚用户目标、约束、验收标准和任务拆解，再让 AI 按规格实现。它把“想法”变成“可检查的开发合同”。

## 典型用例

- 产品经理把功能想法转成可执行 spec。
- 工程团队用 spec 约束 coding agent 的实现边界。
- 训练营把 PRD、任务卡和验收清单做成学习流程。

## 对 AI 产品经理的价值

这是 PM 与 AI 工程工具协作的关键桥梁。PM 不需要先学完整代码，但必须学会写清楚输入、输出、边界条件、验收和风险。

## 对小红书账号的价值

适合讲“文科生转 AI 产品，最该练的是写清楚需求”。用 spec-kit 解释为什么 AI 写代码前需要产品说明书。

## 产品化机会

MVP 可以做“中文 PRD 转 spec 助手”：输入一句需求，输出用户故事、约束、验收标准、任务拆解和给 coding agent 的提示。

## 风险与限制

规格写得差，AI 只会更快地执行错误方向。spec-driven 不能替代需求调研和用户验证。
